01 Overview & goals ▶

Build a single, self-contained JavaScript file that can be dropped onto any client website — either via Google Tag Manager or a direct <script> tag — with zero modification to existing form markup.

The script must capture UTM attribution data and all form field values at submission time, then POST a structured JSON payload to a configurable backend endpoint. It replaces the current per-form webhook approach, which is fragile and requires bespoke installs.

Core goals
  • Zero touch on existing form HTML across all client sites
  • Survive SPA navigation, lazy-loaded content, and dynamically injected forms (HubSpot, etc.)
  • Capture UTMs with last-touch attribution, persisted across sessions via cookie
  • Support native HTML forms, HubSpot embedded forms, Gravity/WP Forms, and Typeform iFrames
  • Deliver payload reliably to a custom backend — with retry on failure
  • Single config object at the top of the file — one line to change per client

03 Script architecture Required ▶

The script is a single IIFE (immediately invoked function expression) with no external dependencies. It must be compatible with all modern browsers and IE11 is explicitly out of scope.

Execution order (on page load)
  • Step 1 — UTM capture: Read UTM params from window.location.search. If any UTM is present, overwrite the attribution cookie immediately (last-touch)
  • Step 2 — Session metadata: Capture document.referrer, page URL, timestamp, and any client-defined custom dimensions
  • Step 3 — Form watcher init: Start MutationObserver on document.body to catch dynamically injected forms
  • Step 4 — Bind existing forms: Find and bind all currently-present forms immediately on load
  • Step 5 — HubSpot listener: Attach window.addEventListener('message', ...) to catch HubSpot's cross-origin postMessage events
  • Step 6 — Typeform listener: Same postMessage approach, filtered by Typeform's known event schema
Module structure inside the IIFE
AttributionTracker/ ├── config // per-client settings object ├── utm.capture() // read + write UTM cookie ├── utm.get() // read current UTM state from cookie ├── forms.bind() // attach submit listeners to native forms ├── forms.observe() // MutationObserver for dynamic injection ├── forms.extract() // serialize form fields to key/value map ├── hubspot.listen() // postMessage listener for HS embed events ├── typeform.listen() // postMessage listener for Typeform events ├── payload.build() // assemble the final JSON payload └── delivery.send() // POST with fetch + retry logic
04 UTM capture & persistence Required ▶
Parameters to capture
utm_source utm_medium utm_campaign utm_term utm_content gclid fbclid msclkid ttclid li_fat_id

All click IDs (gclid, fbclid, etc.) should be captured alongside UTMs as they are essential for ad platform attribution matching.

Attribution logic (last-touch)
  • On every page load, check window.location.search for any of the above parameters
  • If any are found, write ALL current URL params (not just UTMs) into a single cookie named _attr_data, serialized as JSON, with a 30-day expiry
  • This overwrites any previous cookie value (last-touch wins)
  • If no UTM params are present on the current page load, leave the existing cookie untouched
  • Cookie must be set with SameSite=Lax; path=/ — do not set HttpOnly (JS must read it)
Cookie domain: default to current domain. If the config specifies a cookieDomain (e.g. .example.com), use that so attribution persists across subdomains.
Additional session metadata to always capture (not from cookie)
  • page_url — full URL at time of form submission (window.location.href)
  • referrer — document.referrer at time of form submission
  • submitted_at — ISO 8601 timestamp
  • landing_page — first page URL from cookie (set on first visit if not already set, never overwritten)
05 Form detection & field extraction Required ▶
Native HTML forms
  • On load, query document.querySelectorAll('form') and bind a submit event listener to each
  • On submit event, use FormData API to extract all field values (inputs, selects, textareas, checkboxes, radios)
  • Skip fields with type="hidden", type="password", and any field whose name/id matches the config's excludeFields array
  • Capture the form's id, name, and action attributes as form metadata
  • Do NOT call event.preventDefault() — the form must still submit normally
MutationObserver (for dynamic / SPA-injected forms)
  • Observe document.body with { childList: true, subtree: true }
  • On each mutation, check added nodes for <form> elements and bind them if not already bound (use a WeakSet to track already-bound forms)
HubSpot embedded forms
  • Listen to window.addEventListener('message', handler)
  • Filter for events where event.data.type === 'hsFormCallback' and event.data.eventName === 'onFormSubmitted'
  • Extract field data from event.data.data.submissionValues
  • Also capture event.data.id (the HubSpot form GUID) as form metadata
  • Note: HubSpot also fires onFormReady and onFormSubmit — use onFormSubmitted (past tense) as it fires after the submission is complete
Gravity Forms / WPForms
  • These render as standard <form> elements with class gform_wrapper or wpforms-form
  • The native HTML form listener covers basic submissions
  • However, both support AJAX submission that does NOT fire a native submit event — also listen for their JS hooks: gform_confirmation_loaded (Gravity) and wpformsAjaxSubmitSuccess (WPForms) on the document
  • On these AJAX events, re-extract field data from the form element before it's cleared
Typeform (iFrame embed)
  • Typeform runs in an iFrame — direct DOM access is not possible
  • Listen for window.message events where event.data.type === 'form-submit' from Typeform's embed SDK
  • Field-level values are NOT available from Typeform's postMessage by default — capture the responseId and formId from the event payload. Flag this in the payload as form_type: 'typeform' so the backend knows to fetch full response data from the Typeform API if needed
Field exclusion & PII handling
  • Always exclude: password, credit_card, cvv, ssn — match on field name/id/type
  • Config option excludeFields: [] allows additional field name patterns to skip
  • Config option captureFields: [] — if non-empty, only capture fields whose names match this allowlist (whitelist mode)
06 Payload structure Required ▶

Every form submission should POST a single JSON object to the backend. The schema below is the required standard — the backend should be built to accept this shape.

{ "client_id": "acme-corp", // from config "submitted_at": "2025-09-15T14:32:00Z", "page_url": "https://acme.com/contact", "landing_page": "https://acme.com/?utm_source=google", "referrer": "https://google.com", "attribution": { "utm_source": "google", "utm_medium": "cpc", "utm_campaign": "brand-q3", "utm_term": "acme software", "utm_content": "ad-variant-b", "gclid": "EAIaIQ...", "fbclid": null, "msclkid": null }, "form": { "id": "contact-form", // form id attribute or HS guid "name": "Contact Form", // form name attr or derived label "type": "native", // native | hubspot | gravity | wpforms | typeform "action": "/submit" // native forms only }, "fields": { "first_name": "Jane", "last_name": "Smith", "email": "jane@example.com", "company": "Acme Corp", "phone": "+1 555 000 0000", "message": "I'd like a demo" }, "custom": {} // any key/value from config.customDimensions }
The fields object uses the raw form field name attribute as the key. The backend is responsible for any field name remapping to CRM properties.
07 Backend delivery & retry logic Required ▶
Primary delivery — fetch
  • Use fetch(config.endpointUrl, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(payload) })
  • Use keepalive: true on the fetch call — this ensures the request completes even if triggered just before page unload (important for native form submits that navigate away)
  • The backend must return a CORS header allowing the client's origin. This is a backend requirement, not a script concern — document it for the backend developer
Retry logic
  • On network failure or non-2xx response, retry up to 2 times with a 1.5s delay between attempts
  • On final failure, store the payload in localStorage under key _attr_queue as a JSON array
  • On next page load, check _attr_queue and attempt to flush any queued payloads before running the main init
  • On successful flush, remove the entry from the queue
  • Cap the queue at 10 entries to prevent unbounded localStorage growth
Beacon fallback
  • If the form submit navigates the page away and keepalive fetch is not supported (rare), fall back to navigator.sendBeacon() with the JSON payload as a Blob with application/json type
08 Deployment — GTM & direct script Config ▶
Google Tag Manager
  • The script is pasted as a Custom HTML tag in GTM
  • Trigger: All Pages — Page View (or DOM Ready if forms load slowly)
  • Tag firing priority: set to 10 (fires early, before other tags)
  • The config object sits at the top of the Custom HTML block — the only thing an installer needs to edit
  • Do not use GTM variables to pass the endpoint URL — keep everything inside the script for portability
Direct script tag
<script src="https://cdn.yourclient.com/attribution.js"></script> <script> window.AttributionTrackerConfig = { clientId: 'acme-corp', endpointUrl: 'https://api.yourbackend.com/leads', cookieDomain: '.acme.com' }; </script>

When loaded via direct script, expose config via window.AttributionTrackerConfig before the script loads, OR pass config inline at the top of the script file per client.

Script hosting
  • Host the minified script on a CDN (Cloudflare, Fastly, S3+CloudFront)
  • Each client gets the same base script — config is the only difference
  • Optionally: generate a client-specific URL like /attribution/acme-corp.js with config baked in at build time, so installers paste a single tag with no separate config block
09 Config object reference Config ▶
var config = { // REQUIRED clientId: 'acme-corp', // unique per client endpointUrl: 'https://api.yourbackend.com/leads', // your backend POST URL // OPTIONAL — attribution cookieDomain: '.acme.com', // omit to default to current domain cookieName: '_attr_data', // default; change if conflicts exist cookieDays: 30, // default 30 // OPTIONAL — form handling excludeForms: ['#newsletter-form'], // CSS selectors to skip excludeFields: ['phone_ext'], // field name patterns to exclude captureFields: [], // if non-empty: ONLY capture these field names // OPTIONAL — debug debug: false, // if true, console.log all payloads instead of sending // OPTIONAL — custom dimensions // any key/value pairs added here will appear in payload.custom customDimensions: { site_version: 'v2', ab_test: 'control' } };
10 Edge cases to handle Required ▶
  • Multi-step forms: Only fire the payload on final submission — detect by checking whether the submit event leads to a confirmation state vs. advancing a step. For HubSpot, use onFormSubmitted which fires only on true final submit
  • Duplicate binding: Use a WeakSet (or a data-attr-bound attribute) to prevent the same form being bound multiple times by the MutationObserver
  • SPA route changes: If the site uses pushState/replaceState (React, Vue, etc.), re-run the UTM capture step on each route change by patching history.pushState and listening for popstate
  • Cookie blocked (Safari ITP): Safari aggressively expires cookies set via JS. If the cookie cannot be set, fall back to sessionStorage for that session only. Log a warning in debug mode
  • CSP restrictions: The script makes an outbound POST — the client's Content-Security-Policy must allow connect-src to the endpoint domain. Document this requirement for installers
  • Form submit before script loads: If a user submits a form before the script has initialized (very fast user, slow CDN), the event is missed. Mitigate by loading the script in <head> or with high GTM priority. This is an acceptable known gap — document it
  • Multiple forms on one page: All forms are bound independently — each gets its own payload with its own form metadata
  • Form inside an iFrame (non-Typeform): Out of scope. Cross-origin iFrames cannot be accessed. Document this clearly
11 QA & testing checklist Pre-launch ▶
  • Load page with UTMs in URL — verify cookie is written with correct values
  • Load page WITHOUT UTMs — verify existing cookie is not overwritten
  • Clear cookie, load page without UTMs — verify attribution fields are null/empty in payload (not erroring)
  • Submit a native HTML form — verify payload arrives at backend with correct fields and attribution
  • Submit a HubSpot embedded form — verify onFormSubmitted event is captured and payload is sent
  • Submit a Gravity Forms AJAX form — verify gform_confirmation_loaded triggers payload
  • Submit a Typeform — verify postMessage is caught and responseId is in payload
  • Navigate to a new page (SPA), then submit a form — verify forms on the new route are bound
  • Throttle network to Slow 3G, submit form — verify retry logic fires and payload eventually delivers
  • Block the endpoint entirely — verify payload is queued in localStorage
  • Reload page after blocked delivery — verify queued payload is flushed on next load
  • Enable debug: true — verify all payloads log to console and nothing is POSTed
  • Install via GTM — verify tag fires on all pages, no console errors
  • Install via direct script tag — verify same behavior
  • Test on Safari — verify sessionStorage fallback if cookie is blocked
12 Explicitly out of scope Note ▶
  • Writing UTM values back into hidden form fields (not needed — backend receives attribution separately in the payload)
  • Cross-origin iFrame forms other than Typeform (requires postMessage cooperation from the third party)
  • IE11 support
  • CRM field mapping (backend responsibility)
  • De-duplication of submissions (backend responsibility)
  • User identification / stitching across devices
  • Server-side UTM capture
  • A/B test variant tracking beyond what's passed via customDimensions
